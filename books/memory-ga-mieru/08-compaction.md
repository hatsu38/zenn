## 1

木曜の朝。

蓮見の宿題を片付けるべく、オフィスの一角で調べ物をしていた。

「なんで空き区画が、こんなに残るのか」

蓮見の言葉が、ずっと引っ掛かっていた。

ドームで視た光景——歯抜け、虫食い——その風景を、言葉で説明できるようになることが、朝の体操を卒業する条件だった。

ネット記事を漁った。ヒープ、メモリ、Ruby。

そして、ある言葉に行き当たった。

「**断片化**」

読んだ。英語では fragmentation。メモリの領域が小さく分割されて、使える連続空間がない状態。空きの合計はたくさんあるのに、つながった空き地がない。だから、大きな領域を確保できない。

ドームで視ていた光景が、ようやく名前を持った。

視えてから、名前を知る。この順序は、いつも同じだった。

## 2

空き区画が返らない理由は、理解していた。

ページを返すには「丸ごと空」でなければならない。Ruby は、ページという区画に、オブジェクトを並べて置く。その中に、1 つでも生きているオブジェクトがいたら、その区画は返せない。

だから、歯抜けに散らばった生きている光のせいで、ほぼ空のページも、返すことができない。

でも——ずっと、1 つの疑問が胸にあった。

オブジェクトが散らばっているのは、仕方ない。生まれたときに、たまたま空いていた場所に置かれるのだから。でも、そのあとで、引っ越しさせることはできないのか。

生きているオブジェクトを「前の方のページに集める」——そうすれば、後ろのページは、空になるのではないか。

その疑問を携えて、さらに調べ続けた。

## 3

夜。帰宅して、Ruby のドキュメントを追った。

GC（ガベージコレクタ）の細目。世代別 GC。mark-and-sweep。

そして——

「GC.compact」

その言葉を見つけたとき、息が止まった。

読み進めた。

生きているオブジェクトを、メモリの前の方に詰めて引っ越しさせる。空になったページを解放する。

「……動かせるんだ」

呟いた。

オブジェクトは、動かせるんだ。引っ越しさせることができるんだ。

## 4

翌朝。オフィスで、蓮見を捕まえた。

「歯抜けを作って、compact を打つ実験をしたいです」

蓮見が目を上げた。

「compact?」

「GC.compact です。オブジェクトを詰めて、空きページを返すものです」

蓮見が椅子を引いた。

「……面白い。見せてみろ」

オフィスの隅で、新しい Ruby スクリプトを用意した。

```ruby
# compact.rb ── 歯抜けの街と、区画整理
def stats = "live_slots=#{GC.stat[:heap_live_slots]}, pages=#{GC.stat[:heap_allocated_pages]}, RSS=#{(`ps -o rss= -p #{Process.pid}`.to_i / 1024.0).round(1)}MB"

GC.start
puts "開始時:     #{stats}"

data = Array.new(2_000_000) { "resident-#{rand(1000)}" }
puts "200万個:    #{stats}"

keep = []
data.each_with_index { |s, i| keep << s if i % 100 == 0 }  # 100人に1人だけ、歯抜けに残す
data = nil
GC.start
puts "歯抜けGC後: #{stats}"

GC.compact  # 区画整理: 住民を詰めて引っ越しさせる
puts "区画整理後: #{stats}"
```

実行した。

```
開始時:     live_slots=17913, pages=31, RSS=17.1MB
200万個:    live_slots=2413137, pages=1491, RSS=138.5MB
歯抜けGC後: live_slots=37939, pages=1491, RSS=138.7MB
区画整理後: live_slots=37962, pages=998, RSS=107.9MB
```

蓮見が、スクリーンを睨んだ。

しばらく、どちらも黙っていた。

## 5

初めて見た。

ドームの中。視界の片隅で、すうっと光が立ち上がり始めた。

散らばっていた光たち——約 2 万の粒子が、一斉に動く。

列をなして、前へ。前へ。

前方の区画へ、ぞろぞろと引っ越していく。

それに伴って、後ろのほうの——すっかり人口ゼロになった区画が——端から順に、大地に還っていく。

ドームの外殻が——初めて、目に見えて、縮んだ。

pages 1491 から。998。493 の区画が消えた。

RSS 138.7 MB から 107.9 MB。約 31 MB。OS に返った。

「……初めて見たか。縮むところ」

蓮見の声は、ほぼ呟きだった。

でも、その中には、確実な何かが籠もっていた。

## 6

蓮見が、画面から目を離した。

「これが、compact か」

「はい。オブジェクトを詰めて、空いたページを返す。……オブジェクトって、動かせるんですね」

蓮見が、ゆっくり頷いた。

「でも、使いどころは選ぶ。毎回、やるもんじゃない」

「なぜ?」

「住民を引っ越しさせるのは、コストがかかる。特に数百万いたら。それに」蓮見が指を立てた。「動かせないやつもいる」

「動かせない?」

「ページの中には、C 拡張から指をさされているオブジェクトもいる。動かしたら、C の側がアドレスを見失う。だから Rails なら、起動が済んで温まったあとに一度だけ打つ、みたいな使い方が定番だ」

蓮見の説明は短かったが、正確だった。

万能薬ではない。使いどころを選ぶ。

## 7

蓮見が、私を見た。

「で。お前の宿題だ」

「宿題?」

「空き区画が返らない理由。説明できるようになったか」

私は、頷いた。

ドームで視たもの。歯抜け。虫食い。オブジェクトが散らばっていて、ページを丸ごと空にできない状態。それが、断片化。

そして、GC.compact は、その散らばりを直す。オブジェクトを詰めて、空になったページを解放する。

「説明できます」

蓮見が頷いた。

「よし。朝の体操を止めるのに、残る条件は 1 つだ」

「『もう増えない』ことの監視ですね」

「そうだ。『朝になると、毎日、体格が増えている』——その周期を止める。そして、その周期が止まったことを、見張る」

蓮見が立ち上がった。

「来週。監視を組む。その後、朝の体操の cron を消す。やるか」

「やります」

## 8

その夜。帰宅して、手元の Rails のドームを視た。

この街の後ろのほうにも、空き区画は散らばっている。今朝の実験で、「詰められる」ことは分かった。でもあれは、ほんの数秒だけ生きた、実験用の仮の街での話だ。

本物の本番のドームは——今夜も、朝の体操で再起動されて、明日また膨らんでいく。

来週、その周期を止める。

でも、と思う。

視えない人たちは、どうやって、このドームを見守るんだろう。

私の目は、一つしかない。そして、その目に映るものは、私にしか視えない。

私が見張り続けるやり方は、見張りとは呼ばない。数字が、チームの全員に、同じことを教えてくれる形にしないと。

本当の「朝の体操、卒業」は——私がいなくても回る形にしないと、意味がないんだ。

## 今夜の観測メモ

- **断片化** = 空きの合計は十分でも、生きているオブジェクトが散らばっていてページを丸ごと空にできない状態。Ruby のヒープでは、区画に「虫食い」が生じ、ページ単位での返却が止まる
- **GC.compact** = 生きているオブジェクトを前方にまとめて引っ越しさせ、空になったページを解放する操作。実測：pages 1491 → 998（493 区画返却）、RSS 138.7MB → 107.9MB（約 31MB を OS に返却）
- **compact の効果測定** = 前後で `GC.stat[:heap_allocated_pages]` と RSS を比べる。ページ数が減っていれば、断片化が解消されて区画が OS に返った証拠
- **compact は万能薬ではない** = 実行コストがかかる。動かせないオブジェクト（C 拡張から参照されているもの）も存在する。Rails では起動完了後に一度実行するなど、使いどころを選ぶ必要がある
- **残る宿題は監視** = 「もう増えない」ことを、誰にでも見える数字で見張る仕組みが必要。最終話へ
