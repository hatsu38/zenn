## 1

深夜 2 時 40 分。オンコール当番の通知が鳴った。

「[alert] メモリ使用率が 90% を超えています」

私──音無灯、Web エンジニア 3 年目──は、ベッドから起き上がってノート PC を開く。技術記事投稿サービスの Rails アプリケーションだ。記事 10 万件、コメント 100 万件。昨夜も、おととい夜も同じアラートで叩き起こされている。

ダッシュボードを見ると、Puma プロセスのメモリグラフが、右肩上がりで天井に向かっていた。

RSS: 940 MB。

「メモリが足りない」──言葉としては知っていた。だが、それが何を意味するのか、私には実感がなかった。メモリなんて、どこかの見えない場所にあるコンピュータの部品だ。足りないって、どういう状態？ 何をどうすれば足りるの？ その問い詰めたい気持ちより、障害対応の焦りのほうが重かった。

応急処置は決まっている。プロセスを再起動する。

```bash
sudo systemctl restart puma_production
```

コマンドを打つ。Puma が落ちて、数秒後に起動する。メモリグラフは、ドカンと下に落ちて、また一から上昇曲線を描き始める。アラームは鳴り止む。

Slack に報告を上げる。「メモリ枯渇。Puma を再起動して復帰しました。原因は調査予定」。嘘ではないが、本当ではない。原因の見当すら、付いていない。

チームの先輩・蓮見からのコメント。

「で、数字は?」

証拠主義の鬼。蓮見は何かあると必ず数字を要求する。

「再起動直後の RSS は 45 MB です。940 MB から一気に落ちました」

「で、また育つわけだ」と蓮見。「リークの疑いは」

「……未特定です。調査します」

会話は終わった。蓮見は「まあ緊急時だから」という空気だけを残して、スレッドを閉じた。私も仕事に戻った。ふりをした。心の中では、別の話が延々と回転していた。

「メモリ」

それが何かが分からないのに、どうやって調査するんだ。

## 2

翌日の夜。自宅で療養中だった。

昨晩の対応から数時間後、私は過労で意識を失って倒れていた。朝、目が覚めたときには頭がボーッとしていて、会社に電話して「今日は休みます」と言った。医学的には過労性の一時的な意識喪失らしい。怖い。

自宅のデスクに戻った夜、何となく気晴らしに、手元の MacBook で個人開発の Rails アプリを起動してみた。

```bash
cd ~/my_rails_app
rails s
```

いつもと同じコマンド。だが、画面に表示されたとき──世界が変わった。

私の目の前に、ガラスのドーム状の街が出現していた。

透き通った薄緑色のドーム。その中に、淡い光の粒が無数に漂っている。星が並んでいるみたいだ。でも星じゃない。粒たちは、ゆっくり、じっくり、呼吸するように明滅している。粒の大きさはばらばら。米粒みたいに小さいものから、野球ボールくらいのものまで。

何これ？

私は手を伸ばして、モニタに触った。ドームは触れない。触ってもガラスの感触はないが、確かに「そこにある」感覚がある。

息をひそめて、ドームを見た。粒たちは変わらず、呼吸している。

私は正気を失った。そういう結論が出た。

深夜 2 時 40 分のメモリアラートで一徹夜して、その後も頭がフラフラだったから、脳が見せている幻覚だ。目を瞑って深呼吸をする。

10 数えて、目を開ける。

ドームはまだそこにあった。

## 3

幻覚は、消え方がおかしかった。サーバを Ctrl-C で止めると、ドームは砂の城が崩れるように、すっと消えた。起動し直すと、また現れる。同じ形。少しだけ違う、粒の並び。

プロセスを起動するたびに、街がひとつ生まれる。止めるたびに、消える。

「何かのアプリだ」

自分に言い聞かせた。Chrome の拡張機能か何か、勝手にインストールされたやつだろう。

だが、ドームはモニタの枠の外にも見える。枠を超えて、机全体に広がっている。コンソール ウィンドウを動かしても、ドームは動かない。それ自体の位置を持っている。

「正気かな……」

一度、声に出してそう言ってみた。

答える者はいない。ドームの粒たちは、呼吸し続ける。

そのうち、一つの直感が浮かんだ。

「これ、本当なんじゃ……」

もし本当だとしたら。もし私の脳が正常で、これが実在する何かを「視ている」のだとしたら。その「何か」は、きっと数字で説明できる。

私は立ち上がって、調査を始めることにした。

## 4

落ち着け。視えているものが本物かどうかは、数字で確かめられるはずだ。

昨晩のダッシュボードに出ていた数字を思い出す。RSS。あのときは読み飛ばした。調べてみると、Resident Set Size──プロセスが物理メモリ（RAM）に実際に載せている量のことで、単位は KB らしい。

メモリというのは、プロセスの作業机のようなものだ、と解説記事にあった。プロセスは生きている間、コードも、データも、作りかけの計算結果も、ぜんぶメモリの上に広げて仕事をする。プロセスが死ねば、机の上のものはぜんぶ片付けられる。

──さっき、サーバを止めた瞬間にドームが消えたのは、そういうこと？

確かめたい。まず、いちばん小さな Ruby プロセスで試す。エディタで数行のスクリプトを書いた。自分自身の RSS を表示して、すぐに死ぬだけのプロセスだ。

```ruby
# rss.rb
def rss_mb
  (`ps -o rss= -p #{Process.pid}`.to_i / 1024.0).round(1)
end

puts "RSS: #{rss_mb} MB"
```

```bash
ruby rss.rb
```

出力: `RSS: 17.3 MB`

実行した一瞬、机の端に豆粒みたいなドームが生まれて、ぱっと消えた。プロセスの誕生と、死。素の Ruby プロセスは、生まれた時点で 17.3 MB ぶんの机を使っている。

視えているものと、数字。まだ一致とまでは言えない。でも、同じ方向を指している。

## 5

一瞬で消えられると、観察ができない。今度は、消えないプロセスがほしい。対話型の irb なら、私が `exit` と打つまで生き続けるはずだ。

```bash
irb
```

視界の隅に、小さなドームが、ぽん、と生まれた。今度は消えない。プロンプトが私の入力を待っている。つまりこのプロセスは、いま生きている。

さっきの関数を、そのまま打ち込む。

```ruby
def rss_mb
  (`ps -o rss= -p #{Process.pid}`.to_i / 1024.0).round(1)
end

puts "RSS: #{rss_mb} MB"
```

出力: `RSS: 25.9 MB`

さっきのスクリプトは 17.3 MB だったのに、irb は 25.9 MB。同じ Ruby なのに、8 MB あまり体格がいい。irb は対話のための道具──補完とか、履歴とか──を抱えて生きているから、そのぶん机が広い。そういうことだろうか。

言われてみれば、目の前のドームも、さっき一瞬だけ見えた豆粒より、心なしか大きい。

数字と、視えているもの。まだ証明とは呼べない。でも、二つは同じ方向を指し続けている。

調べものの途中で、「Ruby のメモリと言えばまず `GC.stat`」と書いてある記事を見つけた。GC。ガベージコレクション。名前だけは聞いたことがある。打ってみる。

```ruby
pp GC.stat.slice(:count, :minor_gc_count, :major_gc_count, :heap_live_slots, :heap_free_slots, :old_objects)
```

出力:

```
{count: 7,
 minor_gc_count: 5,
 major_gc_count: 2,
 heap_live_slots: 45045,
 heap_free_slots: 8665,
 old_objects: 24371}
```

数字の海。

heap。slots。old_objects。単語は英語として読める。意味は、ひとつも分からない。飛行機のコックピットに迷い込んだ気分だった。計器はぜんぶ本物で、針はぜんぶ動いているのに、どれを見たら飛べるのかが分からない。

「でも、この数字たちは、本当にある」

私が打ったコマンドが返してきた数字だ。捏造ではない。幻覚でもない。

「視えているのは──たぶん、これなんだ」

小さなドーム。その中の、無数の光。この計器盤が読めるようになれば、あの光がなんなのか、いつか言葉で説明できるようになるのかもしれない。

## 6

翌日。会社のチームチャットで、例のメモリ枯渇アラートについて蓮見と雑談していた。

「そういえば、あのアプリ、私が当番じゃない夜もアラート鳴ってません？」

「鳴ってる」と蓮見。「でも 4 時には収まる。『朝の体操』が走るから」

朝の体操。聞き慣れない言葉にチャットを遡ると、それは社内の符牒だった。cron で毎晩 4 時に自動実行される、Puma の再起動。負荷の少ない時間帯に計画的にプロセスを殺して、メモリをリセットする仕組み。

「誰が設定したんですか、これ」

「もう分からない。3 年前にはあった」

「メモリが増え続ける原因って、特定されてます？」

「されてない」蓮見の返信は速かった。「調べた人もいない。再起動で収まるから」

再起動で、収まるから。

モヤモヤが、胸の底に溜まった。毎晩リセットするということは、次の再起動までの 24 時間で、必ず満杯近くまで太り続けるということだ。太る原因は直さず、毎晩のリセットに頼っている。いつか、リセット 1 回では足りなくなる日は来ないんだろうか。

でも、誰もそれを疑問に思っていない。「朝の体操」は、その名の通り朝の習慣のように、今夜も当たり前に実行される。

## 7

その夜、私はもう一度、手元の Rails を起動した。

ドームが生まれる。ブラウザからアプリを開いて、ページをクリックして回る。

クリックするたび、ドームの中で光が一斉に瞬いた。無数の粒が生まれて、渦を巻いて──少しすると、大半はすっと暗くなって消えていく。

でも、全部じゃない。

消えない粒が、少しずつ、残っていく。

30 分もクリックし続けると、はっきり分かった。ドームの中の光は、ゆっくり、確実に増えている。本番のあのグラフと同じだ。右肩上がりの、あの曲線。あれは数字なんかじゃなくて、この光の粒が降り積もっていく姿だったんだ。

「中を、視たい」

粒たちが何なのか。なぜ増えるのか。なぜ、消える粒と、消えない粒があるのか。

でも、外からでは体格の変化しか分からない。中の構造は、すりガラスの向こうみたいに、ぼんやりしている。

「もっと近くで。もっと詳しく」

私は irb に向き直って、読めない計器盤をもう一度開いた。動き続ける針の一本一本に、意味があるはずだった。

（第2話「体の中の群衆」につづく）

## 今夜の観測メモ

物語に出てきた Ruby コマンドはすべて実測値です。あなたの rbenv Ruby 4.0.5 でも再現できます。

- **メモリ（RAM）はプロセスの作業机**。プロセスは生きている間、コードもデータもすべてメモリの上に広げて仕事をし、死ぬと片付けられる
- **RSS (Resident Set Size)** は、プロセスが物理メモリに実際に載せている量。単位は KB。`ps -o rss= -p <PID>` で確認できる
- 素の Ruby 4.0.5 スクリプトの RSS: **17.3 MB**。irb セッションの RSS: **25.9 MB**（対話用の道具のぶん大きい。環境により多少ブレます）
- `GC.stat` には計器がたくさんある。今夜はすべて「読めない計器盤」のまま。これからの話で、一つずつ読めるようにしていく
- 毎晩のプロセス再起動（通称「朝の体操」）は**対症療法**。RSS が増え続ける原因を突き止めるのが本治療だ
