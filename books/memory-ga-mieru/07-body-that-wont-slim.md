## 1

火曜。修正デプロイの日。

朝 9 時、蓮見が新しいコミットを本番へ流した。消し忘れのミドルウェア、削除。たったそれだけ。

デプロイが走って、プロセスが入れ替わる。古いプロセスが静かに終わり、新しいプロセスが起動する。その瞬間——ドームが、新しくなる。

午後、本番のグラフを見た。old_objects の曲線が、完全にフラットになった。波線ではなく、直線。リークは、止まった。

「よし」

蓮見の声は短かったが、その中には確実な満足があった。

オフィスの一角で、小さな達成感が漂っていた。チームスラックでは「朝の体操が終わる日が来たね」というコメントが上がっていた。

でも、夕方——私だけが、小さな違和感を覚えていた。

## 2

その夜、帰宅してから、確かめたいことがあった。

修正が入った世界では、こういうことが起きるはずだ。大量に抱え込まれていたオブジェクトが、ようやく手放される。だったら、「抱えて、手放す」だけの最小のスクリプトで、そのあと体格がどうなるかを、先に見ておける。

```ruby
# diet.rb ── 手放したのに、痩せない
def rss_mb = (`ps -o rss= -p #{Process.pid}`.to_i / 1024.0).round(1)

GC.start
puts "開始時:            RSS=#{rss_mb}MB, live_slots=#{GC.stat[:heap_live_slots]}, pages=#{GC.stat[:heap_allocated_pages]}"

data = Array.new(2_000_000) { "big-object-#{rand(100_000)}" * 5 }
puts "200万個抱えた状態: RSS=#{rss_mb}MB, live_slots=#{GC.stat[:heap_live_slots]}, pages=#{GC.stat[:heap_allocated_pages]}"

data = nil
GC.start
puts "全部手放して GC 後: RSS=#{rss_mb}MB, live_slots=#{GC.stat[:heap_live_slots]}, pages=#{GC.stat[:heap_allocated_pages]}"
```

実行した。

```
開始時:            RSS=17.3MB, live_slots=17934, pages=31
200万個抱えた状態: RSS=384.6MB, live_slots=2230096, pages=5392
全部手放して GC 後: RSS=354.2MB, live_slots=17954, pages=4906
```

ドームを見つめた。

手放した瞬間、200 万の光が波に呑まれて、街は広大な更地に変わった。空き地だらけ。誰もいない。寂しい光景。

でも——ドームの外殻が、縮んでいない。

体が、大きいままだ。街を囲むガラスの外枠——プロセスの体格——が、ほとんど変わっていない。

RSS 384.6 MB から 354.2 MB。差は 30 MB。

30 MB。

200 万の光が全部消えたのに。live_slots は 2230096 から 17954 へ。完全に戻った。それなのに。

「……おかしい」

呟いた。

数字が、私の視ていたドームと、完全に一致している。一致しているから、怖い。

視えているものは正しい。だから——この差が、説明できない。

## 3

翌朝、蓮見の席へ向かった。

「修正が入りました。でも、ドームの体格が、縮みません」

蓮見が顔を上げた。

「で、数字は」

手元で diet.rb を実行した。出力を見せた。

蓮見の目が、数字を追った。

「……live_slots は完全に戻る。RSS は 30 しか下がらない。pages も」蓮見が画面に近づいた。「5392 から 4906。差は 486 ページ。9 割が残ったままだ」

「はい」

蓮見が、ゆっくり後ろに下がった。

「リークは直った。old_objects のグラフも、本番で確認した。増え続けない」

「……ですが」

「でも、体格が縮まない」

蓮見がそう言った。

「何が起きているんだ」

それは、私への質問ではなく、蓮見自身への問いだった。

そして蓮見は、その問いを、こちらに返した。

「調べてこい」

## 4

ドームの中で、光の消えた区画——ページ——を眺めた。

空き家。たくさんの空き家。誰も住んでいない区画。でも、その区画は、街の一部のままだった。

Ruby は、オブジェクトを「ページ」という区画に並べて置く。そのページは、何千何万も、街の中に散在していた。

修正後、200 万のオブジェクトが消えても、pages は 5392 から 4906 へ——ほぼ 9 割が残った。

ということは。

ページは、消えない。空いても、返されない。Ruby は「また使うかもしれない」と思って、空き区画を、ずっと握っていた。

でも、それだけじゃ、説明が足りない。

RSS の 30 MB——その下の層がいる。

## 5

街は、温室だった。ドームの中に並ぶオブジェクトと、その区画（ページ）。その街全体が、もっと大きな「敷地」の上に建っている。

敷地を管理しているのは、malloc という土地管理人。C の世界で、メモリを借りたり返したりするときに間に立つ、仲介役だ。Ruby がページを増やしたいとき、malloc に頼む。malloc は手持ちの敷地から切り出すか、足りなければ、さらにその下の大地——OS——からまとめて借りてくる。

OS が malloc に貸している土地の総面積。それが、RSS だった。ダッシュボードで見ていた、あの数字。

ということは、「返らない」は、二段階で起きている。

まず、Ruby が空き区画を手放さない。「また使うかもしれない」と、更地になったページを握り続ける（pages が 9 割残ったのが、それだ）。Ruby が握っている限り、その土地は malloc から見れば貸出中のままだ。

そして、Ruby が本当に返した分すら、malloc は「どうせまた借りに来る」と、大地にすぐには返さない。

だから、RSS は 30 MB しか下がらない。街（Ruby のヒープ）は空き地だらけなのに、大地への返却は、ほんの端っこだけ。

「……土地、返していないんだ」

呟いた。

この街は、痩せてないんじゃない。——土地を返していないだけなんだ。

## 6

蓮見に報告した。

三層構造。Ruby のヒープ。malloc の敷地。OS の大地。

蓮見は、黙って聞いていた。

「リークは、確かに直った」蓮見が言った。「old_objects は増え続けない。それで十分だ」

「……でも」

「お前が気にしてるのは、RSS の絶対値だ。でも、見るべきはそこじゃない」

蓮見が指を立てた。

「傾きだ」

「傾き?」

「グラフの傾き。RSS が増え続けるか、止まるか。止まれば病気じゃない。高止まりでも、増えなけりゃ問題ない。大事なのは『これからどうなるか』だ」

その言葉は、正確だった。

健康とは、絶対的な体格ではなく——変化しないこと。安定していること。

「で」蓮見が続けた。「朝の体操を止めるには、『もう増えない』ことの監視と、もう一つ」

「もう一つ?」

「なんで空き区画が、こんなに残るのか。説明がつくようになっとけ」

蓮見の目が、画面の数字をもう一度追った。

「9 割が残る。そこまで多く残る理由が、お前の中で、腑に落ちていないはずだ」

その通りだった。

## 7

その夜。帰宅して、もう一度 diet.rb を流し、手放したあとの街を、じっくり眺めた。

街は、本当に、空き家だらけだった。

誰も住んでいない区画が、9 割。

でも、空き家は、まとまって空いてはいなかった。

ぽつん。ぽつん。

ほとんど空になった区画にも、生きている光が一つ、二つ、置き去りのように残っている。空き家と住人が、同じ区画に混ざっている。——歯抜け。虫食い。

だから、区画ごとは、返せない。

「区画ごと返したくても、返せないんだ……」

つぶやいた。

なんで、こんなに歯抜けなんだろう。

そこにこそ、次の謎がいた。

## 今夜の観測メモ

- **三層構造** = Ruby のヒープ（オブジェクトが並ぶ）→ malloc が管理する敷地（ページをまとめた領域）→ OS から借りた大地。RSS は「OS が貸した土地の総面積」を示す指標
- **オブジェクトが消えても区画は返らない** = live_slots（住民）は 2230096 → 17954 と完全に戻るのに、RSS は 384.6MB → 354.2MB と 30MB しか下がらない。Ruby が空いたページを「また使うかも」と手放さず、malloc も OS に土地をほぼ返さないため
- **ページの動き** = heap_allocated_pages は 5392 → 4906 へ。9 割以上のページが、空いたまま街に残る
- **メモリの健康は絶対値ではなく傾き** = RSS が「高い」ことは病気ではない。大事なのは「増え続けるか」。修正後、増え続けない限り、朝の体操は卒業できる
- **空き区画がなぜ返せないのか** = 次話へ。生きているオブジェクトと死んだ区画が混在して散らばり、ページごと返せない理由が、そこにある
