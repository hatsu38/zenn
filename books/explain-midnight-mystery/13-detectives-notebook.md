## 1

一週間後の、午前1時52分。

Slack が鳴った。

『[alert] 著者検索 API のレスポンスタイムが 3 秒を超えています』

僕はベッドから起き上がり、ノート PC を開いた。心臓は、先週ほど跳ねていなかった。

レプリカに接続。スロークエリログから該当クエリを回収。頭に付けるのは、もう迷わない。

```sql
EXPLAIN (ANALYZE, BUFFERS) SELECT ...;
```

出力、14 行。読む。上からじゃない──**インデントの一番深いところから**。

目が最初に止まったのは `Filter:` の行だった。あるはずのインデックスが使われず、Seq Scan。Filter の中に `(id)::text = ...` ──**キャスト**。

「……人相書き 7 号。化けの皮キャストだ」

昨日リリースされた検索機能のコードを開く。新しく入ったメンバーが、ID をクエリパラメータから**文字列のまま**流し込んでいた。悪意も過失もない、ORM の静かなすり替え。アプリの型を直す PR を作り、レビュー依頼を投げ、応急のホットフィックスを当てた。グラフが凪ぐ。

時計を見た。午前2時14分。──先週、僕が桐生さんに電話をかけた、ちょうどその時刻だった。

桐生さんの Slack に、一行だけ送っておいた。

『事件解決。今夜は起こさずに済みました。詳細は朝の #explain-捜査資料 で』

翌朝、返信は一言だけだった。

『悪くない』

## EXPLAIN 捜査手帳

### 【定石】打ち方はこれ一択

```sql
EXPLAIN (ANALYZE, BUFFERS) SELECT ...;
```

- `EXPLAIN` 単体は**実行しない**（本番でも安全）。`ANALYZE` を付けると**実行される**
- 更新系に打つなら必ず `BEGIN; ... ROLLBACK;` で包む（第2話）

### 【読む順序】5 ステップ

1. **ノード名** ─ どんな読み方・結合か（第1・3・6・7話）
2. **actual time** ─ 実測ミリ秒。A ≪ B は「1 行目が早い」型、A ≒ B は「全行揃うまで待つ」型（第2・5話）
3. **Buffers** ─ hit / read の比率。時間はブレるがページ数はブレない（第8話）
4. **rows vs actual rows** ─ 乖離が捜査の起点（第2・9話）
5. **loops** ─ 内側ノードは `× loops` が真の仕事量（第2・6話）

プラン木は**インデントの深い子から**動く。子が親に行を流す（第7話の Hash が好例）。

### 【最初に見る 3 つの数字】

1. **Execution Time** ─ 全体の実時間
2. **Buffers: shared hit / read** ─ I/O が原因か
3. **actual rows × loops** ─ どのノードが本当に働かされたか

### 【乖離の目安】rows vs actual rows

| 比率 | 判定 | 動き方 |
|---|---|---|
| 2 倍以内 | 誤差 | 何もしない |
| 10 倍以上 | プラン選択ミスの疑い | 統計・相関カラム・LIKE を疑う |
| 100 倍以上 | 暴走の疑い | `ANALYZE` を打つ。`pg_stats` を覗く |

### 【ノード人相一覧】

| ノード | 動き | 強い場面 / 弱い場面 |
|---|---|---|
| Seq Scan | 全ページ連続読み | 大量ヒット ◎ / 一部だけ欲しい ✕ |
| Index Scan | 索引 → ヒープへ 1 件ずつ | 数行ピンポイント ◎ / 中規模 ✕ |
| Index Only Scan | 索引だけで完結 | `Heap Fetches: 0` なら本領。大きければ名前だけ |
| Bitmap Heap Scan | 候補をまとめてページ順読み | 中規模ヒット ◎ |
| Nested Loop | 外側 × 内側ループ | 外側小＋内側索引 ◎ / 外側の推定ミスで暴走 ✕ |
| Hash Join | 小側を名簿化→大側を照合 | 両方大きい ◎ / `Batches > 1` で劣化 |
| Merge Join | ソート済み 2 本を並走 | 両キーに B-tree ◎ |
| Sort | 全行並べ替え | A ≒ B 型。`external merge Disk` は work_mem 溢れ |
| Limit | 上から N 件で打ち切り | A の小さいノードと好相性 |
| Memoize | 内側の結果をキャッシュ | Cache Hits の比率で効きを読む |
| Gather | 並列の集約 | loops = 並列数に注意 |

### 【単価表と手計算】

| パラメータ | 既定値 | 意味 |
|---|---|---|
| seq_page_cost | 1.0 | 連続読み 1 ページ（基準） |
| random_page_cost | 4.0 | ランダム読み 1 ページ（SSD なら 1.1〜2.0 に下げる余地） |
| cpu_tuple_cost | 0.01 | 1 行処理 |
| cpu_operator_cost | 0.0025 | 演算子 1 回 |

- Seq Scan: `1.0 × relpages + 0.01 × reltuples`（第1話。articles なら `11181 + 1000 = 12181`）
- Sort: `2 × 0.0025 × N × log2(N)` ＋溢れたら一時ファイル I/O（第5話）

### 【困ったときの武器庫】

| 状況 | 武器 |
|---|---|
| 推定が現実とズレてる | `ANALYZE <table>;`（約 3 万行サンプリング・低負荷。第9話） |
| 大量更新の後始末 | `VACUUM ANALYZE <table>;`（通常 VACUUM はロックしない。第10話） |
| プランナの 2 位以下を見たい | `SET enable_* = off`（診断器具。本番常用は劇薬。第7・11話） |
| 統計の中身を見たい | `pg_stats` / `pg_stat_user_tables`（last_autoanalyze, n_dead_tup。第9・10話） |
| N+1 を探したい | `pg_stat_statements` の calls 降順（第12話） |
| プランを固定したい | **下から検討**: autovacuum 調整 → CREATE STATISTICS → MATERIALIZED / OFFSET 0 → pg_hint_plan（第11話） |

### 【人相書きの 3 つの目】（第12話）

1. `Filter:` に**関数・キャスト**が見える → 式インデックス / 型合わせ
2. **Sort** や意外な **Seq Scan** がいる → 索引でソートを消す / SELECT * をやめる / LIKE を見直す
3. **read が異常に大きい / calls が異常に多い** → count の設計 / N+1 を疑う

### 【心得】

- EXPLAIN は実行しない。ANALYZE は実行される。**鞘（BEGIN/ROLLBACK）を忘れるな**
- インデックスが使われないのは、たいてい**正しい入札の結果**
- プランナは嘘をつかない。**古い真実を読まされることがあるだけ**だ
- コスト最小 = 実時間最小は、**推定が正しいときだけ**成り立つ
- 深夜の障害に悪人はいない。全員が仕様どおりに動いて、組み合わせが壊れる。**だから、しくみを 1 本の線で読め**

## 3

投稿には、思ったより多くのリアクションが付いた。例の新しいメンバーからは『7 号、私です……穴があったら入りたい』という自首と、真剣な質問がいくつも飛んできた。夕方には誰かが『#explain-道場』というチャンネルを作っていた。

定時後、廊下で桐生さんとすれ違った。

「捜査手帳、見た」

「あ……どうでした」

「1 か所直せ。『プランナは嘘をつかない』の行──あれは俺の台詞じゃなくて、**お前があの夜、自分の手で確かめたこと**だ。受け売りみたいな顔で書くな」

それだけ言って、桐生さんはエレベーターに消えた。褒め言葉だと理解するのに、数秒かかった。

その夜、僕は自分の PC のサンドボックスで、意味もなく EXPLAIN を打った。

```sql
EXPLAIN SELECT * FROM articles;
                            QUERY PLAN
------------------------------------------------------------------
 Seq Scan on articles  (cost=0.00..12181.00 rows=100000 width=852)
```

`12181.00`。

11,181 ページの連続読みと、10 万行の CPU 処理。もう呪文には見えない。1 の位まで、自分の電卓で出せる数字だ。

午前2時14分に鳴る通知音を、怖くないと言えば嘘になる。でも今は知っている。画面の向こうでどれだけ複雑な事件が起きていても──**嘘をつかない捜査資料が、必ず 1 枚残っている**ことを。

（了）
